# PageIndex 项目中文说明文档

## 一、项目概述

**PageIndex** 是由 [Vectify AI](https://vectify.ai) 开源的**无向量化、基于推理的 RAG（检索增强生成）系统**。

与传统基于向量数据库的 RAG 不同，PageIndex 不依赖向量相似度搜索，而是通过构建文档的**层级树状索引**（类似"目录"），再利用 LLM（大语言模型）对该索引进行**推理式检索**，模拟人类专家阅读和定位文档内容的方式。

**本项目已扩展为 Web 可视化系统**，提供完整的文档上传、树结构可视化和智能问答功能。详细使用说明请参考 [使用说明.md](使用说明.md)。

### 核心特点

| 特性 | 说明 |
|------|------|
| 无向量数据库 | 不需要向量数据库，使用文档结构 + LLM 推理进行检索 |
| 无分块 | 文档按自然章节组织，而非人为切块 |
| 类人检索 | 模拟人类专家浏览文档、提取知识的方式 |
| 可解释性强 | 检索基于推理过程，可追踪、可解释，附带页码和章节引用 |
| **多格式支持** | 支持 PDF、Markdown、TXT、JSON、CSV、Word 等格式 |
| **Web 可视化** | Vue3 前端 + FastAPI 后端，提供完整的交互界面 |

### 工作原理（两步走）

1. **生成树状索引**：将文档转换为语义化的层级树结构（类似增强版目录）
2. **推理式检索**：通过 LLM 在树结构上进行推理搜索，找到最相关的文档章节

## 三、核心模块详解

### 3.1 `run_pageindex.py` — 主入口

命令行入口脚本，支持两种文档类型的处理：

- **PDF 处理**：`python run_pageindex.py --pdf_path /path/to/file.pdf`
- **Markdown 处理**：`python run_pageindex.py --md_path /path/to/file.md`

#### 可选参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--model` | `gpt-4o-2024-11-20` | 使用的 OpenAI 模型 |
| `--toc-check-pages` | `20` | 检查目录的页数（仅 PDF） |
| `--max-pages-per-node` | `10` | 每个节点最大页数（仅 PDF） |
| `--max-tokens-per-node` | `20000` | 每个节点最大 token 数（仅 PDF） |
| `--if-add-node-id` | `yes` | 是否添加节点 ID |
| `--if-add-node-summary` | `yes` | 是否添加节点摘要 |
| `--if-add-doc-description` | `no` | 是否添加文档描述 |
| `--if-add-node-text` | `no` | 是否添加节点原文 |
| `--if-thinning` | `no` | 是否进行树剪枝（仅 Markdown） |
| `--thinning-threshold` | `5000` | 剪枝最小 token 阈值（仅 Markdown） |
| `--summary-token-threshold` | `200` | 生成摘要的 token 阈值（仅 Markdown） |

输出结果保存至 `./results/{文件名}_structure.json`。

### 3.2 `pageindex/page_index.py` — PDF 核心处理

该文件是 PDF 处理的核心逻辑，主要功能包括：

1. **目录检测**：检查 PDF 前 N 页是否存在目录（TOC）
2. **标题验证**：通过 LLM 验证标题是否出现在对应页面
3. **页码映射**：将逻辑页码映射到物理页码
4. **树结构构建**：将平面目录列表转换为层级树
5. **节点摘要生成**：为每个节点生成内容摘要
6. **无目录处理**：对没有目录的 PDF，通过 LLM 自动分段

核心函数 `page_index_main(pdf_path, opt)` 是整个处理流程的入口。

### 3.3 `pageindex/page_index_md.py` — Markdown 处理

处理 Markdown 文件的模块，功能包括：

- 基于 `#` 标题层级提取节点
- 构建层级树结构
- 可选的树剪枝（合并过小的节点）
- 可选的节点摘要生成

### 3.4 `pageindex/utils.py` — 工具函数库

提供底层支持功能（**已适配 DeepSeek API**）：

| 类别 | 函数 | 说明 |
|------|------|------|
| LLM 调用 | `ChatGPT_API()` / `ChatGPT_API_async()` | 同步/异步调用 OpenAI 兼容 API（支持自定义 base_url） |
| PDF 操作 | `extract_text_from_pdf()` / `get_page_tokens()` | PDF 文本提取与 token 统计 |
| Token 计算 | `count_tokens()` | 使用 tiktoken 计算 token 数（支持非 OpenAI 模型） |
| 树操作 | `list_to_tree()` / `write_node_id()` / `get_nodes()` | 树结构构建与遍历 |
| JSON 处理 | `extract_json()` / `get_json_content()` | 从 LLM 响应中提取 JSON |
| 日志 | `JsonLogger` | JSON 格式日志记录器 |
| 配置 | `ConfigLoader` | YAML 配置文件加载器 |

**DeepSeek API 适配**：
- 支持通过 `API_BASE_URL` 环境变量配置自定义 API 端点
- `count_tokens()` 对非 OpenAI 模型自动回退到 `cl100k_base` 编码器

### 3.5 `pageindex/config.yaml` — 默认配置

```yaml
model: "deepseek-chat"            # 默认模型（已改为 DeepSeek）
toc_check_page_num: 20            # 检查目录的页数
max_page_num_each_node: 10        # 每个节点最大页数
max_token_num_each_node: 20000    # 每个节点最大 token 数
if_add_node_id: "yes"             # 是否添加节点 ID
if_add_node_summary: "yes"        # 是否添加节点摘要
if_add_doc_description: "no"      # 是否添加文档描述
if_add_node_text: "no"            # 是否添加节点原文
```

### 3.6 Web 系统新增模块

#### `server/services/converter_service.py` — 格式转换服务

将多种文件格式转换为 Markdown，再走统一的 `md_to_tree` 处理管道：

| 格式 | 转换策略 |
|------|---------|
| **TXT** | 自动检测中文章节标题（第X章/节）和英文标题，转为 Markdown 层级 |
| **JSON** | 嵌套对象转标题层级，数组转表格+详情 |
| **CSV** | 自动检测分隔符，生成表格概览+逐行详情 |
| **Word** | 提取 Heading 样式转 Markdown 标题，保留加粗斜体，表格转 MD 表格 |

#### `server/services/chat_service.py` — RAG 问答管道

实现两阶段推理式检索：

1. **树搜索阶段**：将树骨架（不含原文）+ 问题发送给 LLM，获取相关节点列表
2. **答案生成阶段**：提取节点原文，结合问题生成答案（流式输出）

#### `web/src/` — Vue3 前端

- **views/DocumentView.vue**：文档详情页，左侧树可视化 + 右侧问答面板
- **components/tree/TreeViewer.vue**：树容器，支持搜索、展开/折叠、高亮同步
- **components/chat/ChatPanel.vue**：问答界面，SSE 流式输出
- **stores/documents.js & chat.js**：Pinia 状态管理
- **composables/useSSE.js**：通用 SSE 解析器（支持 POST）

## 四、输出格式

生成的树结构 JSON 示例：

```json
{
  "title": "Financial Stability",
  "node_id": "0006",
  "start_index": 21,
  "end_index": 22,
  "summary": "The Federal Reserve ...",
  "nodes": [
    {
      "title": "Monitoring Financial Vulnerabilities",
      "node_id": "0007",
      "start_index": 22,
      "end_index": 28,
      "summary": "The Federal Reserve's monitoring ..."
    }
  ]
}
```

每个节点包含：
- `title`：章节标题
- `node_id`：节点编号（四位数字）
- `start_index` / `end_index`：起止页码
- `summary`：章节内容摘要
- `nodes`：子节点列表（递归结构）

## 六、快速启动指南

### 方式一：Web 可视化系统（推荐）

#### 第 1 步：安装依赖

```bash
# Python 依赖
pip install --upgrade -r requirements.txt
pip install python-docx

# 前端依赖
cd web
npm install
```

#### 第 2 步：配置环境变量

在项目根目录创建 `.env` 文件：

```env
CHATGPT_API_KEY=sk-你的DeepSeek_API_Key
API_BASE_URL=https://api.deepseek.com
```

> 如果使用 OpenAI，则 `API_BASE_URL` 留空或删除该行。

#### 第 3 步：启动系统

双击运行 `start.bat`，或在命令行执行：

```bash
start.bat
```

系统会自动启动：
- **后端服务**：http://localhost:8001
- **前端服务**：http://localhost:3000

在浏览器打开 http://localhost:3000 即可使用。

**详细使用说明请参考 [使用说明.md](使用说明.md)**。

---

### 方式二：命令行模式（原始方式）

#### 第 1 步：安装依赖

```bash
pip install --upgrade -r requirements.txt
```

#### 第 2 步：配置 API Key

在项目根目录创建 `.env` 文件：

```env
CHATGPT_API_KEY=你的API_Key
API_BASE_URL=https://api.deepseek.com  # 可选
```

#### 第 3 步：运行

处理 PDF 文件：
```bash
python run_pageindex.py --pdf_path tests/pdfs/q1-fy25-earnings.pdf
```

处理 Markdown 文件：
```bash
python run_pageindex.py --md_path /path/to/your/document.md
```

结果将输出到 `./results/` 目录下。

## 七、Web 系统功能特性

### 1. 多格式文档上传

支持以下格式：
- **PDF**：学术论文、技术手册、财务报告
- **Markdown**：技术文档、博客文章
- **TXT**：纯文本文档、小说
- **JSON**：API 响应、配置文件、结构化数据
- **CSV**：数据表格、统计报表
- **Word (.docx)**：商务文档、报告、合同

### 2. 树结构可视化

- 📂 层级展示：按文档章节层级显示
- 🔍 搜索过滤：实时搜索节点标题
- 🎨 深度着色：不同层级用不同颜色标识
- ✨ 高亮同步：问答时自动高亮相关节点
- 🔄 展开/折叠：一键展开或折叠所有节点

### 3. 智能问答（RAG）

**两阶段推理式检索**：
1. **树搜索**：LLM 在树骨架上推理，找到相关节点
2. **答案生成**：提取节点原文，生成答案（流式输出）

**特色功能**：
- 💭 思考过程可见
- 🎯 节点引用标注
- 🔗 树同步高亮
- 📜 对话历史保留

### 4. LLM 配置

支持配置：
- API Key
- API Base URL（支持 OpenAI 兼容接口）
- 模型名称（如 `deepseek-chat`、`gpt-4o` 等）
